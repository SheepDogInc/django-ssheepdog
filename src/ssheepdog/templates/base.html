Directions

  • If someone should be temporarily denied access to all Logins, de-select is_active for the user (in the admin) and click Synchronize all. This may be appropriate, for example, for contract or seasonal employees.

  • When someone permanently leaves an organization, disable or delete the user and, in the case of a user who might have had access to the SSHeepdog database, also click Generate new application key.

  • You can also de-select is_active for a Login or a Machine if you wish to prevent inadvertent updates or access to the related Logins. Access can later be re-activated on demand.

  • You can safely click Generate new application key at any time. The application keeps track of which Logins it has successfully upgraded to the new key.

  • It's usually safe to click Synchronize all at any time. If a Login fails to sync, here is a checklist of possible reasons:

    1. The Machine is off-line.
    2. SSH is not configured properly on the Machine.
    3. The .ssh directory for the Login does not have the correct permissions.
    4. This application's public key has not been installed in the .ssh/authorized_keys file in the Login account. Copy and paste the one from clicking on the appropriate Login above.
