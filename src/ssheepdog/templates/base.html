Directions

  • When someone is temporarily denied access to all machines, de-select is_active for the user (in the admin) and Synchronize all. This may be appropriate, for example, for contract or seasonal employees.

  • When someone permanently leaves an organization, disable or delete the user and generate a new application key. This is especially important if the person who left is a developer who had access to the SSHeepdog database.

  • You can also de-select is_active for a login or a machine if you wish to prevent inadvertent updates or access to the machine. Access can later be re-activated on demand.

  • It's usually safe to click Synchronize all at any time. If a machine fails sync, here is a checklist of possible reasons:

    1. The machine is taken off-line
    2. SSH is not configured properly on the machine
    3. The .ssh directory for the login does not have the correct permissions.
    4. This application's public key has not been installed on the machine. Copy and paste the one from clicking on the appropriate login above.
